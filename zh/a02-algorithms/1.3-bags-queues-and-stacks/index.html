1.3 Bags, Queues, and Stacks

0 terms 术语

Interface: description of data type, basic operations.

Implementation: actual code implementing operations. (implementation can’t know details of client needs –> many clients can re-use the same implementation.)

Client: program using operations defined in interface. (client can’t know details of implementaion –> client has many implementation from which to choose.)

1 stacks 栈

1.0 features 特征

A pushdown stack (or just a stack) is a collection that is based on the last-in-first-out (LIFO) policy.

  • LIFO (Last in first out)
  • push item on the top; pop item from the top
  • When a client iterates through the items in a stack with the foreach construct, the items are processed in the reverse of the order in which they were added.

1.2 array implementation

优点:

  • The items in the array are in their insertion order.
  • push and pop operations take time independent of the stack size.

缺点:

  • 需要提前设定数组大小，且不能更改 –> resizing
  • 只支持String数据类型 –> generic
  • 不支持迭代(iteration)
public class FixedCapacityStackOfStrings {
	private String[] a; // stack entries
	private int N; // size
    
	public FixedCapacityStackOfStrings(int capacity)
	{ a = new String[capacity]; }

    public boolean isEmpty() { return N == 0; }
    public int size() { return N; }

    public void push(String item) { a[N++] = item; }
    
	// loitering: Holding a reference to an object when it is no longer needed. 
    public String pop() {
        String item = a[--N];
        a[N] = null; // Avoid loitering
        return item;
    }
}

2 resizing arrays 调整数组大小

解决问题:

  • A client that chooses a large capacity risks wasting a large amount of memory.
  • Every client risks overflow if the collection grows larger than the array.

做法:

  • push(): double size of array a[] when array is full.
  • pop(): halve size of array s[] when array is one-quarter full.

时间和空间消耗:

  • array is between 25% and 100% full.

  • Starting from an empty stack, any sequence of M push and pop operations takes time proportional to M.

  • Uses between ~ 8 N and ~ 32 N bytes to represent a stack with N items. (~8N when full; ~32N when one-quarter full)

ps: slides p25 讨论了 resizing array 与 linked list 的优劣，以及内存消耗

// Move stack of size N <= capacity to a new array of size capacity.
private void resize(int capacity) {
	String[] copy = new String[capacity];
	for (int i = 0; i < N; i++) {copy[i] = a[i]; }
	a = copy;
}

// Add item to top of stack.
public void push(String item) { 
	if (N == a.length) {resize(2*a.length); }
    a[N++] = item;
}

// Remove item from top of stack.
public String pop() {
	String item = a[--N];
	a[N] = null;
	if (N > 0 && N == a.length/4) {resize(a.length/2); }
	return item;
}

3 queues 队列

A FIFO queue (or just a queue) is a collection that is based on the first-in-first-out (FIFO) policy.

  • FIFO (first in first out)
  • enqueue(): insert item at end of queue; dequeue(): remove item from front of queue
  • When a client iterates through the items in a queue with the foreach construct, the items are processed in the order they were added to the queue.

3.2 linked-list implementation

优点:

  • the space required is proportional to the number of items in the collection.
  • the time required per operation is independent of the size of the collection.

缺点:

  • 只支持String数据类型 –> generic
  • 不支持迭代(iteration)
  • 要考虑特殊情况：1. 当queue为空的时候，在末尾(last)添加第一个元素，first和last实际上指向同一个node; 2. 如果只剩下一个元素了，从开头(first)移除该元素，要把last所指的也设置为null

linked-list的一个问题:

  • 开头(first) 支持添加和移除元素的操作，末尾(last) 支持添加元素，但从末尾移除元素的操作会很麻烦，因为要通过遍历，才能找到倒数第二个node (解决办法: doubly-linked lsit)
public class LinkedQueueOfStrings {
	private Node first;
    private Node last;
    private N; // size
	
    // nested class to define nodes
    private class Node {
        String item;
        Node next;
    }
    
	public boolean isEmpty() {return first == null; }
	public int size() {return N;}
    
    public void enqueue(String item) {
		Node oldlast = last;
		last = new Node();
		last.item = item;
		last.next = null;
		if (isEmpty()) {first = last; } // special case for empty queue
		else {oldlast.next = last; }
        N++;
	}

    public String dequeue() {
		String item = first.item;
		first = first.next;
		if (isEmpty()) {last = null;} // special cases for empty queue
        N--;
		return item;
	}
}

5 iteration 迭代

Challenge: support iteration over stack/queue items by client without revealing the internal representation of the stack.

Java solution: make stack/queue implement the java.lang.Iterable interface.

What is an Iterable?

  • Has a method that returns an Iterator. 可以重写，使用 Iterable 内的 iterator() 方法
public interface Iterable<Item> {
    Iterator<Item> iterator(); // returns an Iterator
}

What is an Iterator?

  • Has methods hasNext() and next(). 可以重写，使用 Iterator 内的方法，来实现 foreach 功能

We have to include import java.util.Iterator at the beginning of the program because Iterator is not part of java.lang (even though Iterable is part of java.lang).

We always use an empty method for remove(), because interleaving iteration with operations that modify the data structure is best avoided.

public interface Iterator<Item> {
    boolean hasNext();
    Item next();
    void remove(); // optional; use at your own risk
}

Why make data structures Iterable?

  • Java supports elegant client code. (the clear and compact code that is free from dependence on the details of a collection’s implementation.)
// "foreach" statement(shorthand)
Stack<String> stack = new Stack<String>();
for (String s : stack){
    StdOut.println(s);
}

// equivalent code(longhand)
Iterator<String> i = stack.iterator();
while (i.hasNext()) {
    String s = i.next();
    StdOut.println(s);
}

6.2 arithmetic expression evaluation

计算 (1 + ( ( 2 + 3 ) * ( 4 * 5 ) ) )

首先，不能省略括号: (1 + ( ( 2 + 3 ) * 4 * 5 ) ) )

然后，需要两个stack:

  1. operand/value stack
  2. operator stack

Two-stack algorithm [E. W. Dijkstra]

  • Operand/Value: push onto the value stack
  • Operator: push onto the operator stack.
  • Left parenthesis: ingore.
  • Right parenthesis: pop operator and two values; push onto the operand/value stack the result of applying that operator to those operands/values.

引申发现

Observation 1: Dijkstra’s two-stack algorithm computes the same value if the operator occurs after the two values.

( 1 ( ( 2 3 + ) ( 4 5 * ) * ) + )

Observation 2: All of the parentheses are redundant!

1 2 3 + 4 5 * * +

7.2 Randomized Queue

在array中随机放入以及随机取出元素

import edu.princeton.cs.algs4.StdRandom;
import java.util.Iterator;
import java.util.NoSuchElementException;

public class RandomizedQueue<Item> implements Iterable<Item> {
    private Item[] arr;
    private int size;

    // construct an empty randomized queue
    public RandomizedQueue() {
        arr = (Item[]) new Object[2];
        size = 0;
    }

    // is the randomized queue empty?
    public boolean isEmpty() {return size == 0; }

    // return the number of items on the randomized queue
    public int size() {return size; }

    // arrange the size of the array
    private void changeSize(int n) {
        Item[] newArr = (Item[]) new Object[n];
        for (int i = 0; i < size; i++) {newArr[i] = arr[i]; }
        arr = newArr;
    }

    // add the item
    public void enqueue(Item item) {
        if (item == null) {throw new IllegalArgumentException(); }
        if (size == arr.length) {changeSize(2 * arr.length); }
        if (size == 0) {
            arr[size++] = item;
            return;
        }
        int randomX = StdRandom.uniform(size);
        Item x = arr[randomX];
        arr[randomX] = item;
        arr[size++] = x;
    }

    // remove and return a random item
    public Item dequeue() {
        if (isEmpty()) {throw new NoSuchElementException(); }
        if (size == arr.length / 4) {changeSize(arr.length /2); }
        int randomX = StdRandom.uniform(size);
        Item x = arr[randomX];
        arr[randomX] = arr[--size];
        arr[size] = null;
        return x;
    }

    // return a random item (but do not remove it)
    public Item sample() {
        if (isEmpty()) {throw new NoSuchElementException(); }
        return arr[StdRandom.uniform(size)];
    }

    private class ArrayIterator implements Iterator<Item> {
        private int t;
        private final int[] randomXs;

        public ArrayIterator() {
            t = 0;
            randomXs = new int[size];
            for (int i = 0; i < size; i++) {
                randomXs[i] = i;
            }
            StdRandom.shuffle(randomXs);
        }

        @Override
        public boolean hasNext() {return t < size; }

        @Override
        public Item next() {
            if (!hasNext()) {throw new NoSuchElementException(); }
            return arr[randomXs[t++]];
        }

        @Override
        public void remove() {throw new UnsupportedOperationException(); }
    }

    // return an independent iterator over items in random order
    @Override
    public Iterator<Item> iterator() {return new ArrayIterator(); }

    // unit testing
    public static void main(String[] args) {
        RandomizedQueue<Integer> queI = new RandomizedQueue<Integer>();
        for (int i = 0; i < 7; i++) {queI.enqueue(i); }
        System.out.println(queI.size());
    }
}
